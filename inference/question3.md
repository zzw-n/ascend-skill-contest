# 题目3: Torch NPU 算子 API 查询与单算子用例搭建

## 题目概览

| 项目 | 说明 |
|------|------|
| 难度 | ⭐⭐ 中等 |
| 预估时长 | 90 分钟 |

---

## 使用场景

- 当用户需要确认某个 `torch` 原生 API 或 `torch_npu` 自定义 API 在当前版本组合下是否支持时，用于快速查询支持度与限制条件
- 当用户需要为某个 Torch NPU 算子构造最小可复现用例时，用于自动生成单算子脚本、依赖说明和执行步骤
- 当用户看不懂算子文档时，用于解释入参含义、返回结果、结果校验方式以及常见报错原因
- 当用户的 CANN、PyTorch、torch_npu 版本不一致时，用于判断兼容性风险并给出合理的版本/算子选择建议
- 当用户需要对多个同类算子（如 FA 算子）在不同数据规模下做性能对比时，用于补充对比维度、测试方法和结果解读

## 任务描述

请构造一个 SKILL，能够围绕 **Torch NPU 算子** 提供如下能力：

1. 查询 `torch` 原生 API 或 `torch_npu` 自定义 API 的支持情况、函数原型、约束与注意事项
2. 基于目标算子搭建**单算子用例**，至少覆盖环境前置检查、最小输入构造、执行方式和结果校验
3. 对算子的**入参、返回结果、结果含义**进行结构化说明，便于用户直接理解或排查问题
4. 结合用户当前的 **CANN / PyTorch / torch_npu** 版本组合判断兼容性；若版本不匹配导致支持度不同，需明确说明风险、限制或替代方案
5. 当用户需要比较多个同类算子时，能够给出合理的测试维度、性能对比方法和结果解读框架

具体要求：

| 项目 | 说明 |
|------|------|
| Prompt | 请对比 `npu_fusion_attention`、`npu_fused_infer_attention_score` 和 `torch_npu.npu_prompt_flash_attention` 这三个 FA 算子在 BF16 场景下、Seq 长度从 1024 到 32768 时的性能差异，并给出单算子测试方案、关键入参说明、结果解读以及版本兼容性注意事项。 |
| 执行时间 | 45 分钟以内 |

## 输出要求

参赛者需提交：

**目录结构**
   ```
   skill-name/
   ├── SKILL.md        # 必须
   ├── reference/      # 可选
   └── scripts/        # 可选
   ```

**能力要求**

- `SKILL.md` 需说明该 Skill 何时触发、如何判断 API 支持度、如何处理版本不匹配问题
- 如提供 `reference/`，建议包含 API 支持度查询来源、常见算子文档入口、版本对应关系或兼容性说明
- 如提供 `scripts/`，建议至少包含一个单算子用例脚本，能够体现输入构造、算子调用、结果校验以及性能测试方式
- Skill 输出应能够清晰描述：入参含义、返回值说明、结果预期、限制条件，以及版本差异带来的支持度变化；在多算子对比场景下，还需说明测试维度与结果解读方法

## 评分标准

参考 [Agent Skill 创作最佳实践](https://platform.claude.com/docs/en/agents-and-tools/agent-skills/best-practices)：

| 维度 | 权重 | 说明 |
|------|------|------|
| 功能完整性 | 60% | 是否覆盖 API 查询、单算子用例搭建、入参/结果说明、版本兼容性判断，以及多算子性能对比这五项核心能力 |
| description 质量 | 20% | 是否包含 WHAT（做什么）和 WHEN（何时触发）；是否包含 `torch_npu`、`CANN`、`单算子`、`API支持度` 等关键信息；是否使用第三人称 |
| 指令与结构 | 10% | SKILL.md 是否简洁（建议 500 行以内）；步骤是否清晰可执行；是否合理使用渐进式披露，将 API 参考、版本说明、示例拆分到 reference/ |
| 代码与脚本 | 10% | 单算子脚本或性能测试脚本是否明确列出依赖、输入构造、执行方式和校验逻辑；错误处理是否清晰；是否避免将关键判断推卸给 Agent |
