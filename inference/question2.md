# 题目2: 推理服务化性能摸测

## 题目概览

| 项目     | 说明     |
| -------- | -------- |
| 难度     | ⭐ ⭐ 中等 |
| 预估时长 | 90分钟   |

---

## 使用场景

- 对于服务化推理测试不熟悉的新人，用于自动化进行推理服务化性能测试
- 在需要进行大量性能数据测试，寻优，整理测试报告的情况下，使用agent自动化进行，解放人力。

## 任务描述

使用vLLM benchmark / ais bench （二选一），对已存在的推理服务进行性能测试。

具体要求：

1. 版本：

   - vLLM benchmark：使用0.13.0版本
   - AISBench：使用最新版本

   假设已存在该版本可用工具（可使用conda环境/镜像/容器）。

2. 支持多模态测试，需与用户交互确认测试方式。

3. 支持多种数据集，需与用户交互确认使用数据集。若数据集不存在，agent可自行在线下载。

4. 开始测试前与用户交互确认测试参数（输入输出token数，采样参数，发送速率等）。

5. 支持一次测试多个case（多个服务，多种输入输出）

### 使用示例

| 项目     | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| Prompt   | 测试 xxx 服务（提供测试端口）在满足TTFT < 2s，TPOT < 50ms 情况下的最优吞吐。 |
| 交互     | 向用户确认可用环境，测试数据集，输入输出长度等。             |
| 输出结果 | 成功完成寻优，并输出完整测试报告与结果分析。                 |

## 评分标准

参考 [Agent Skill 创作最佳实践](https://platform.claude.com/docs/en/agents-and-tools/agent-skills/best-practices)：

| 维度             | 权重 | 说明                                                         |
| ---------------- | ---- | ------------------------------------------------------------ |
| 功能完整性       | 60%  | 是否能成功引导完成服务化性能测试，并成功找出最优并发，覆盖关键步骤 |
| description 质量 | 225% | 是否包含 WHAT（做什么）和 WHEN（何时触发）；是否具体、含关键词；是否用第三人称 |
| 指令与结构       | 10%  | SKILL.md 是否简洁（建议 500 行以内）；指令步骤是否清晰可执行；是否合理使用渐进式披露 |
| 代码与脚本       | 10%  | 脚本是否明确列出依赖；路径是否使用正斜杠；错误处理是否清晰；是否避免推卸给 Agent |
